# كتالوج الخدمات الاستشارية

تطبيق ويب تفاعلي لاستعراض وإدارة **بطاقات الخدمات الاستشارية** ضمن *برنامج الاستشارات الرقمية 2026* — يعرض 47 خدمة موزّعة على 7 قطاعات ومراكز، مصنّفة حسب الأهداف الاستراتيجية والفئات ومراحل التحول الرقمي الثلاث.

> 🔒 **بيانات داخلية.** الكتالوج مشفّر بالكامل (AES‑256‑GCM). الموقع علني لكن البيانات لا تُقرأ إلا بعد إدخال كلمة المرور الصحيحة في المتصفح.

---

## المزايا

- **لوحة رئيسية (Dashboard):** مؤشرات سريعة، توزيع المراحل (مخطط دائري)، الخدمات حسب القطاع، وبطاقات القطاعات والأهداف والفئات — كلها قابلة للنقر للتصفية.
- **بطاقات الخدمات:** عرض شبكي أنيق مع شارات القطاع والمرحلة والمالك و SLA.
- **بطاقة تفصيلية:** لوحة جانبية تعرض كل حقول الخدمة (الوصف، الأهداف، المتطلبات، المخرجات، المستفيدون، مبرر تصنيف المرحلة…).
- **فلترة غنية بالأزرار والبطاقات** (بدل القوائم المنسدلة): حسب القطاع، الإدارة العامة، المرحلة، الهدف، الفئة، المستفيدين، المالك، والممثل — مع بحث نصي فوري.
- **إضافة / تعديل / حذف** للخدمات، وإدارة قوائم: الإدارات العامة، ملاك الخدمات، وممثلي الخدمات (إعادة تسمية تنعكس على كل الخدمات).
- **حسابات مستخدمين وأدوار:** مدير النظام (صلاحية كاملة + الموافقة على الحسابات والتعديلات) ومالك/ممثل خدمات (يرى ويعدّل خدمات قطاعه فقط، وأي تعديل يمر عبر مراجعة المدير قبل التطبيق).
- **تاريخ آخر تحديث** يُحدَّث تلقائيًا مع كل تعديل.
- **مظهر فاتح/داكن**، تصميم متجاوب، خط IBM Plex Sans Arabic، ودعم كامل للاتجاه من اليمين لليسار.
- **نسخ احتياطي**: تصدير/استيراد نسخة JSON، وتحديث مباشر من المستودع.

## آلية العمل — «قاعدة بيانات على GitHub»

- تُخزَّن البيانات مشفّرة في الملف `data/catalog.enc` داخل المستودع (نص مشفّر فقط، لا بيانات صريحة أبدًا).
- **طبقتان من الدخول:**
  1. **كلمة مرور الكتالوج** (مشتركة) — يدخلها أي عضو ليُفكّ تشفير البيانات في متصفحه ويرى الكتالوج.
  2. **حساب شخصي** (فردي) — بعد فتح الكتالوج، يسجّل كل شخص دخوله بحسابه الخاص (أو يطلب حسابًا جديدًا)، وعلى أساسه تُحدَّد صلاحياته:
     - **مدير النظام:** يرى ويعدّل كل شيء مباشرة، ويوافق على الحسابات الجديدة والتعديلات المُرسلة من الآخرين، ويعيّن الأشخاص كملاك/ممثلين لخدمات محددة.
     - **مالك/ممثل خدمات:** يرى ويعدّل خدمات **قطاعه فقط**؛ أي إضافة/تعديل/حذف يرسلها يذهب لقائمة "طلبات المراجعة" (🔔 في الشريط العلوي) ولا يُطبَّق على الكتالوج المشترك إلا بعد موافقة مدير النظام.
- **أول شخص يفتح الكتالوج بعد النشر يُنشئ حساب مدير النظام الأول تلقائيًا** (لا يوجد بعد أي مستخدمين).
- صلاحية الكتابة في GitHub (رمز وصول بصلاحية `Contents: Read and write`) **مُضمّنة داخل نفس الملف المشفّر نفسه** — لا تظهر أبدًا في كود الصفحة العلني. عند حفظ أي تغيير (بما فيها الحسابات والتعديلات المعلّقة) تُعاد كتابة الملف المشفّر في المستودع عبر GitHub Contents API.

> ⚠️ **حدود هذا النموذج:** الأدوار والصلاحيات مُطبَّقة في واجهة المستخدم (JavaScript) لا في خادم حقيقي — لأن الموقع مُستضاف بالكامل كملفات ثابتة (GitHub Pages) بلا أي خادم خلفي. أي شخص يعرف كلمة مرور الكتالوج ويملك خبرة تقنية (أدوات المطوّر في المتصفح) يمكنه تجاوز هذه القيود تقنيًا. هذا مقبول لأداة داخلية موجّهة لفريق موثوق، لكنه **ليس حماية أمنية صارمة** — لا تُستخدم هذه الآلية لبيانات حسّاسة تتطلب تحكّمًا صارمًا بالوصول.

### تغيير كلمة المرور
من **الإعدادات → كلمة مرور الكتالوج** (لمدير النظام فقط). تُعاد تشفير البيانات بالكامل (بما فيها رمز الكتابة وكل الحسابات) وتُحفظ تلقائيًا. **أبلغ الفريق بالكلمة الجديدة.**

### تدوير رمز الكتابة (Rotate)
عند الحاجة لتغيير الرمز المُضمّن (مثلًا بعد انتهاء صلاحيته)، دون الحاجة لملف البيانات الصريح:
```bash
node tools/set-write-token.js data/catalog.enc "<كلمة المرور>" "<الرمز الجديد>"
git add data/catalog.enc && git commit -m "تدوير رمز الكتابة" && git push
```

## التطوير المحلي

```bash
# أي خادم ملفات ثابت، مثلاً:
npx serve .
# ثم افتح http://localhost:3000
```
> يجب التشغيل عبر خادم HTTP (وليس فتح الملف مباشرة) حتى يعمل جلب البيانات.

## إعادة توليد البيانات المشفّرة

```bash
node tools/build-data.js <catalog.json> "<كلمة المرور>" data/catalog.enc --write-token="<رمز الكتابة>"
```
حيث `catalog.json` ملف البيانات الصريح (لا يُرفع للمستودع مطلقًا — مُستثنى في `.gitignore`). معامل `--write-token` اختياري، ويُستخدم فقط عند إعادة البناء الكامل من الصفر — لتدوير الرمز في نسخة منشورة استخدم `tools/set-write-token.js` بدلًا من ذلك.

## البنية

| الملف | الوظيفة |
|------|---------|
| `index.html` | الهيكل وتحميل الخطوط والسكربتات |
| `assets/styles.css` | نظام التصميم (فاتح/داكن، RTL) |
| `js/config.js` | الإعدادات، التصنيفات، الألوان، بيانات المستودع |
| `js/crypto.js` | التشفير AES‑256‑GCM / PBKDF2 (متصفح + Node) |
| `js/charts.js` | المخطط الدائري (SVG) |
| `js/icons.js` | مجموعة الأيقونات (SVG) |
| `js/app.js` | الحالة، طبقة GitHub، العرض، الفلاتر، البطاقة التفصيلية، نطاق رؤية القطاع |
| `js/app-ui.js` | التفاعل، الحسابات والأدوار، مراجعة التعديلات، الإضافة/التعديل/الحذف، الإعدادات، شاشة القفل |
| `tools/build-data.js` | أداة بناء الملف المشفّر من البيانات الصريحة |
| `tools/set-write-token.js` | أداة تضمين/تدوير رمز الكتابة داخل ملف مشفّر موجود |
