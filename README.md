# كتالوج الخدمات الاستشارية

تطبيق ويب تفاعلي لاستعراض وإدارة **بطاقات الخدمات الاستشارية** ضمن *برنامج الاستشارات الرقمية 2026* — يعرض 47 خدمة موزّعة على 7 قطاعات ومراكز، مصنّفة حسب الأهداف الاستراتيجية والفئات ومراحل التحول الرقمي الثلاث.

> 🔒 **بيانات داخلية.** الكتالوج مشفّر بالكامل (AES‑256‑GCM). الموقع علني لكن البيانات لا تُقرأ إلا بعد إدخال كلمة المرور الصحيحة في المتصفح.

---

## المزايا

- **لوحة رئيسية (Dashboard):** مؤشرات سريعة، توزيع المراحل (مخطط دائري)، الخدمات حسب القطاع، وبطاقات القطاعات والأهداف والفئات — كلها قابلة للنقر للتصفية.
- **بطاقات الخدمات:** عرض شبكي أنيق مع شارات القطاع والمرحلة والمالك و SLA.
- **بطاقة تفصيلية:** لوحة جانبية تعرض كل حقول الخدمة (الوصف، الأهداف، المتطلبات، المخرجات، المستفيدون، مبرر تصنيف المرحلة…).
- **فلترة غنية بالأزرار والبطاقات** (بدل القوائم المنسدلة): حسب القطاع، الإدارة العامة، المرحلة، الهدف، الفئة، المستفيدين، المالك، والممثل — مع بحث نصي فوري.
- **إضافة / تعديل / حذف** للخدمات، وإدارة قوائم: الإدارات العامة، ملاك الخدمات، وممثلي الخدمات (إعادة تسمية تنعكس على كل الخدمات).
- **تاريخ آخر تحديث** يُحدَّث تلقائيًا مع كل تعديل.
- **مظهر فاتح/داكن**، تصميم متجاوب، خط IBM Plex Sans Arabic، ودعم كامل للاتجاه من اليمين لليسار.
- **نسخ احتياطي**: تصدير/استيراد نسخة JSON، وتحديث مباشر من المستودع.

## آلية العمل — «قاعدة بيانات على GitHub»

- تُخزَّن البيانات مشفّرة في الملف `data/catalog.enc` داخل المستودع (نص مشفّر فقط، لا بيانات صريحة أبدًا).
- **العرض:** يفتح أي عضو الرابط ويدخل كلمة المرور → يُفكّ التشفير في متصفحه.
- **التحرير:** يتطلب **رمز وصول GitHub** بصلاحية `Contents: Read and write`؛ عند الحفظ تُعاد كتابة الملف المشفّر في المستودع مباشرة. يُحفظ الرمز في متصفح المحرِّر فقط.

### تفعيل وضع التحرير
1. أنشئ رمزًا من: **GitHub → Settings → Developer settings → Fine‑grained tokens** ([رابط مباشر](https://github.com/settings/personal-access-tokens/new)).
2. الصلاحية: **Repository access** = هذا المستودع، **Permissions → Contents = Read and write**.
3. في التطبيق: **الإعدادات → وضع التحرير** → ألصق الرمز → «حفظ الرمز».

### تغيير كلمة المرور
من **الإعدادات → كلمة مرور الكتالوج** (يتطلب وضع التحرير). تُعاد تشفير البيانات بالكامل وتُحفظ. **أبلغ الفريق بالكلمة الجديدة.**

## التطوير المحلي

```bash
# أي خادم ملفات ثابت، مثلاً:
npx serve .
# ثم افتح http://localhost:3000
```
> يجب التشغيل عبر خادم HTTP (وليس فتح الملف مباشرة) حتى يعمل جلب البيانات.

## إعادة توليد البيانات المشفّرة

```bash
node tools/build-data.js <catalog.json> "<كلمة المرور>" data/catalog.enc
```
حيث `catalog.json` ملف البيانات الصريح (لا يُرفع للمستودع مطلقًا — مُستثنى في `.gitignore`).

## البنية

| الملف | الوظيفة |
|------|---------|
| `index.html` | الهيكل وتحميل الخطوط والسكربتات |
| `assets/styles.css` | نظام التصميم (فاتح/داكن، RTL) |
| `js/config.js` | الإعدادات، التصنيفات، الألوان، بيانات المستودع |
| `js/crypto.js` | التشفير AES‑256‑GCM / PBKDF2 (متصفح + Node) |
| `js/charts.js` | المخطط الدائري (SVG) |
| `js/icons.js` | مجموعة الأيقونات (SVG) |
| `js/app.js` | الحالة، طبقة GitHub، العرض، الفلاتر، البطاقة التفصيلية |
| `js/app-ui.js` | التفاعل، الإضافة/التعديل/الحذف، الإعدادات، شاشة القفل |
| `tools/build-data.js` | أداة بناء الملف المشفّر |
